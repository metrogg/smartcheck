# SmartCheck UI/交互设计文档

本文件为正式交付用的 UI/交互设计说明，覆盖登录、晨检、员工管理、记录、报表导出和设置。

## 1. 产品目标

- 面向食堂晨检场景，完成“刷脸验温 -> 手部双面识别 -> 身体不适确认 -> 生成晨检记录”的闭环流程。
- 管理端可维护员工与健康证信息、查看/编辑晨检记录、导出报表。

## 2. 信息架构 (IA)

- 登录页
- 首页（主入口）
  - 我要晨检
  - 员工管理
  - 晨检记录
  - 报表导出
  - 设置
- 我要晨检
- 员工管理
  - 员工列表（分页）
  - 员工详情（支持编辑）
  - 新增员工
- 晨检记录（筛选 + 详情查看/编辑）
- 报表导出
- 设置

## 3. 页面级详细设计

### 3.1 登录页

页面结构：

- 顶部标题：登录页标题（可在设置中修改）
- 背景图：登录页背景图（可配置）
- 登录卡片：账号、密码、登录按钮
- 辅助信息：错误提示、版本号

交互：

- 成功登录 -> 进入首页
- 失败提示：账号/密码错误

### 3.2 首页（主入口）

页面结构：

- 顶部栏：
  - 左侧：欢迎语“欢迎进入某某食堂，每日晨检，健康上岗!”
  - 右侧：用户头像 + 设置按钮
- 左侧卡片：晨检步骤说明（固定文案）
  1) 刷脸验温
  2) 手部双面识别
  3) 身体不适确认
  4) 生成晨检记录
- 右侧功能入口卡片/按钮：
  - 我要晨检
  - 员工管理
  - 晨检记录
  - 报表导出

交互：

- 点击进入对应模块
- 头像可进入设置或显示管理员信息

### 3.3 我要晨检

页面结构（左右分栏）：

- 左侧：实时摄像头区域
  - 阶段1：人脸识别（刷脸验温）
  - 阶段2：手部识别（双面切换）
- 右侧上：识别信息卡片
  - 头像、姓名
  - 体温
  - 健康证有效期
  - 健康证剩余天数（<7 天高亮提示）
- 右侧中：手部结果区
  - 正面图 / 反面图
  - 异物检测结果提示
- 右侧下：提交按钮 + 当前流程进度条
- 预留语音提示（未来接入）：UI 上预留“语音提示状态小组件”

交互流程：

1. 人脸识别成功 -> 展示人员信息（头像、姓名、健康证有效期、体温）
2. 自动切换到手部摄像头
3. 采集正面 -> 显示正面图
4. 采集反面 -> 显示反面图
5. 异物检测结果显示在右侧
6. 进入“身体不适确认”（弹窗或右侧卡片）
7. 点击提交 -> 生成晨检记录
8. 成功后自动返回待机首页

异常处理：

- 未识别到人脸：提示“请正对摄像头”
- 体温异常：提示标红
- 手部未检测成功：提示“请伸手至识别区域”
- 健康证过期：标红并强提示
- 权限异常：提示授权

### 3.4 员工管理（列表）

页面结构：

- 顶部搜索栏：姓名/编号
- 列表区域：分页展示
  - 每项显示：头像、姓名、健康证剩余天数、状态颜色
- 右上角：新增员工按钮（+）

交互：

- 分页切换
- 点击员工 -> 进入员工详情

### 3.5 员工详情（支持编辑）

展示字段：

- 基本信息：姓名、编号、身份证
- 人脸照片
- 健康证照片
- 健康证起止日期 + 剩余天数

编辑操作：

- “编辑”按钮 -> 进入编辑模式
- 可修改：姓名、编号、身份证、健康证日期/照片
- 保存后更新

### 3.6 新增员工

输入字段：

- 姓名、编号、身份证
- 人脸照片（调用人脸摄像头拍摄）
- 健康证照片（调用手部摄像头拍摄）
- 健康证起止日期（日期选择器）
- 自动计算健康证剩余天数并显示

交互：

- 必填字段未完成禁止提交
- 提交成功自动回员工列表

### 3.7 晨检记录

筛选条件：

- 日期筛选
- 姓名筛选
- 提问手部情况
- 健康证状态
- 其他身体不适

列表展示字段：

- 姓名、日期、体温
- 手部情况（正常/异常）
- 健康证状态（有效/即将过期/已过期）
- 其他身体不适（有/无）
- 结果状态（通过/不通过）

操作：

- 查看详情
- 编辑记录（支持修正备注/状态）

### 3.8 报表导出

- 支持按筛选条件导出
- Excel 文件格式
- 导出状态提示（导出中/成功/失败）

### 3.9 设置

可配置内容：

- 管理员姓名
- 登录账号
- 登录密码
- 食堂名称
- 当前版本（只读）
- 登录页标题
- 登录页背景图

交互：

- 支持修改并保存
- 修改密码需确认

## 4. 全局交互规范

- 流程进度条：固定显示在“我要晨检”界面
- 健康证预警：剩余 < 7 天黄色提示，过期红色提示
- 晨检完成：自动跳转待机首页
- 语音提示：预留 UI 状态位，后续接入语音播报模块

## 5. 数据结构建议（记录表）

- 姓名
- 编号
- 日期
- 体温
- 手部情况
- 健康证状态
- 其他身体不适
- 结果状态
- 备注

## 6. 可扩展/可升级预留

- 流程扩展：ProgressStepper 可配置，新增步骤仅改配置。
- 识别流程状态机以“枚举 + 事件”驱动，便于插入语音提示/扫码等步骤。
- 员工数据源抽象为 Repository，便于接入云端同步或第三方系统。
- 报表导出抽象服务，支持 Excel/PDF/CSV 切换。
- CameraPanel 可支持双摄/单摄/外接摄像头。
- Theme Token 化（颜色/字体/间距/圆角），便于品牌升级或不同屏幕适配。
