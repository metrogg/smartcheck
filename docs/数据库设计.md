# 一、设计原则

## 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **第一范式（1NF）** | 每个字段都是原子的，不可再分 |
| **第二范式（2NF）** | 主键唯一，非主键字段完全依赖主键 |
| **第三范式（3NF）** | 非主键字段之间不能有依赖关系 |

## 1.2 业务原则

| 原则 | 说明 |
|------|------|
| **主键** | 用自增 ID（简单、可读） |
| **外键** | 有关系的数据要建立关联 |
| **索引** | 经常查询的字段要建索引 |
| **软删除** | 不物理删除，用 isActive 标记 |
| **字段类型** | 合理选择类型（Long 存时间戳） |

# 三、DAO 设计

## 3.1 UserDao

```kotlin
@Dao
interface UserDao {
    
    /**
     * 获取所有活跃用户（带索引）
     */
    @Query("SELECT * FROM users WHERE isActive = 1")
    fun getAllActiveUsers(): Flow<List<UserEntity>>
    
    /**
     * 根据 ID 获取用户
     */
    @Query("SELECT * FROM users WHERE id = :userId")
    suspend fun getUserById(userId: Long): UserEntity?
    
    /**
     * 根据工号获取用户（工号已加唯一索引）
     */
    @Query("SELECT * FROM users WHERE employeeId = :employeeId")
    suspend fun getUserByEmployeeId(employeeId: String): UserEntity?
    
    /**
     * 插入用户
     */
    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insertUser(user: UserEntity): Long
    
    /**
     * 更新用户
     */
    @Update
    suspend fun updateUser(user: UserEntity)
    
    /**
     * 删除用户（软删除）
     */
    @Query("UPDATE users SET isActive = 0 WHERE id = :userId")
    suspend fun softDeleteUser(userId: Long)
    
    /**
     * 物理删除用户（谨慎使用）
     */
    @Query("DELETE FROM users WHERE id = :userId")
    suspend fun deleteUser(userId: Long)
}
```

## 3.2 RecordDao

```kotlin
@Dao
interface RecordDao {
    
    /**
     * 获取最近记录（带 limit）
     */
    @Query("SELECT * FROM check_records ORDER BY checkTime DESC LIMIT :limit")
    fun getRecentRecords(limit: Int = 100): Flow<List<RecordEntity>>
    
    /**
     * 获取某员工的记录
     */
    @Query("SELECT * FROM check_records WHERE userId = :userId ORDER BY checkTime DESC")
    fun getRecordsByUser(userId: Long): Flow<List<RecordEntity>>
    
    /**
     * 按时间范围查询
     */
    @Query("SELECT * FROM check_records WHERE checkTime >= :startTime AND checkTime <= :endTime ORDER BY checkTime DESC")
    fun getRecordsByTimeRange(startTime: Long, endTime: Long): Flow<List<RecordEntity>>
    
    /**
     * 🆕 获取某员工今日是否已检查
     * 使用复合索引 (userId, checkTime)
     */
    @Query("""
        SELECT * FROM check_records 
        WHERE userId = :userId 
        AND checkTime >= :todayStart 
        AND checkTime < :todayEnd
        LIMIT 1
    """)
    suspend fun getTodayRecordByUser(
        userId: Long, 
        todayStart: Long, 
        todayEnd: Long
    ): RecordEntity?
    
    /**
     * 根据 ID 获取记录
     */
    @Query("SELECT * FROM check_records WHERE id = :recordId")
    suspend fun getRecordById(recordId: Long): RecordEntity?
    
    /**
     * 插入记录
     */
    @Insert
    suspend fun insertRecord(record: RecordEntity): Long
    
    /**
     * 更新记录
     */
    @Update
    suspend fun updateRecord(record: RecordEntity)
    
    /**
     * 删除旧记录
     */
    @Query("DELETE FROM check_records WHERE checkTime < :beforeTime")
    suspend fun deleteOldRecords(beforeTime: Long)
    
    /**
     * 删除所有记录
     */
    @Query("DELETE FROM check_records")
    suspend fun deleteAllRecords()
}
```

# 五、设计优化说明

## 5.1 已添加的优化

| 优化项 | 说明 |
|--------|------|
| employeeId 唯一索引 | 防止重复工号，加速查询 |
| isActive 索引 | 软删除查询加速 |
| userId 索引 | 按员工查询记录加速 |
| checkTime 索引 | 按时间范围查询加速 |
| 复合索引 | userId + checkTime 加速今日检查查询 |

## 5.2 冗余存储

以下字段在 check_records 表中冗余存储：

| 字段 | 原因 |
|------|------|
| userName | 方便查询，不必每次关联 users 表 |
| employeeId | 方便查询，不必每次关联 users 表 |

**权衡**：略微增加存储空间，换取查询性能提升。
