# SmartCheck UI 视觉规范与样式指南

本文件用于统一视觉语言与组件样式，保证易用性与可扩展性。

## 1. 视觉方向

- 关键词：卫生、可信赖、清爽、专业。
- 视觉基调：明亮背景 + 蓝/绿为主色，橙色用于预警，红色用于风险。
- 信息层级：主流程更突出，辅助信息弱化。

## 2. 颜色体系（建议）

基础色：

- Primary：#1E6BD6（主色，按钮/关键强调）
- PrimaryDark：#1555A8（强调/按下状态）
- Accent：#2FBF71（通过/成功）
- Warning：#F5A524（预警/即将过期）
- Error：#E5484D（失败/异常）
- Neutral-100：#F7F9FC（背景）
- Neutral-200：#EDF2F7（分割/卡片边界）
- Neutral-600：#5B6675（次级文本）
- Neutral-900：#1B2430（主文本）

状态色规则：

- 通过：Accent
- 即将过期（<7天）：Warning
- 过期/异常：Error

## 3. 字体与排版

- 中文字体：优先使用系统默认（避免引入新字体依赖）。
- 标题层级：
  - H1：24sp，SemiBold
  - H2：20sp，SemiBold
  - H3：18sp，Medium
- 正文：14-16sp，Regular
- 辅助信息：12-13sp，Regular

## 4. 间距与布局

- 页面外边距：12-16dp
- 卡片内边距：12dp
- 组件间距：8-12dp
- 列表项高度：56-72dp
- 输入框高度：48dp

## 5. 圆角与阴影

- 卡片圆角：12dp
- 小组件圆角：8dp
- 按钮圆角：10-12dp
- 阴影：轻量（避免过重投影）

## 6. 组件样式规范

按钮：

- Primary Button：实心主色，白色文字
- Secondary Button：描边/浅底
- 禁用状态：降低透明度 + 灰色文本

卡片：

- 白底 + 轻边界或浅阴影
- 关键结果卡片允许使用状态色背景

列表项：

- 左侧头像/图标
- 中间主信息
- 右侧状态标签/剩余天数

进度条：

- 采用分段式 Stepper
- 当前步骤高亮，其余浅色

## 7. 关键页面视觉要点

首页：

- 左侧步骤说明卡片用浅背景，右侧入口按钮更大。
- 欢迎语强对比，保持温度感。

我要晨检：

- 摄像头区域占比大，结果卡片清晰。
- 健康证过期/体温异常采用显著颜色提示。
- 提交按钮固定在右下，易触达。

员工管理：

- 列表清晰可扫视，分页保持简洁。
- 剩余天数显著但不喧宾夺主。

晨检记录：

- 筛选区和列表区区分明显。
- 通过/未通过状态色明确。

设置：

- 表单分组清晰，强调可配置项。

## 8. 动效建议（轻量）

- 页面进入：淡入 200-300ms
- 步骤切换：短暂高亮/进度条移动
- 提示弹窗：轻量缩放 + 淡入

## 9. 可扩展性注意

- 颜色与字体统一通过 Theme Token 管理。
- 新增页面或流程尽量复用现有组件样式。
- 大屏与小屏使用同一套比例系统缩放。
