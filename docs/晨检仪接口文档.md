# 智能晨检仪 API 接口文档

## 基础信息

- **服务器地址**: `http://设备IP:8080`
- **协议**: HTTP/HTTPS
- **数据格式**: JSON
- **编码**: UTF-8

---

## 认证方式

所有接口（除登录外）需要在请求头中携带 Token：

```
Authorization: Bearer {token}
```

Token 通过登录接口获取，有效期 24 小时。

---

## 通用响应格式

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

### 错误响应

```json
{
  "code": 1001,
  "message": "错误描述",
  "data": null
}
```

### 错误码说明

| 错误码 | 说明 |
|-------|------|
| 0 | 成功 |
| 1001 | 未授权（Token 无效或过期） |
| 1002 | 禁止访问 |
| 1003 | 资源不存在 |
| 1004 | 参数错误 |
| 1005 | 服务器内部错误 |
| 1006 | Token 已过期 |

### 1. 登录接口

获取访问 Token。

**请求**

```http
POST /api/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "123456"
}
```

**响应**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "expiresIn": 86400,
    "tokenType": "Bearer",
    "user": {
      "id": 1,
      "username": "admin",
      "name": "管理员"
    }
  }
}
```

### 2. 获取员工列表

获取所有员工信息。

**请求**

```http
GET /api/employees?page=1&pageSize=20&employeeId=001&isActive=true
Authorization: Bearer {token}
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| page | int | 否 | 页码，默认1 |
| pageSize | int | 否 | 每页条数，默认20，最大100 |
| employeeId | string | 否 | 员工工号，精确匹配 |
| isActive | boolean | 否 | 是否在职（true/false） |

**响应**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "name": "张三",
        "employeeId": "001",
        "idCardNumber": "110101199001011234",
        "healthCertStatus": "VALID",
        "healthCertStartDate": 1704067200000,
        "healthCertEndDate": 1735689600000,
        "faceImage": "/api/employee-images/face_001.jpg",
        "healthCertImage": "/api/employee-images/cert_001.jpg",
        "isActive": true,
        "createdAt": 1704067200000
      }
    ],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "total": 10,
      "totalPages": 1
    }
  }
}
}
```

**健康证状态说明**

| 状态 | 说明 |
|------|------|
| VALID | 有效 |
| EXPIRING_SOON | 即将过期（7天内） |
| EXPIRED | 已过期 |
| NOT_FOUND | 未找到 |

### 6. 增量同步接口

用于避免重复拉取数据，只获取上次同步后的新记录。

**请求**

```http
GET /api/records/sync?lastRecordId=1000&limit=100
Authorization: Bearer {token}
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| lastRecordId | long | 是 | 上次同步的最后一条记录ID |
| limit | int | 否 | 拉取条数，默认100，最大500 |

**响应**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1001,
        "userId": 1,
        "userName": "张三",
        "employeeId": "001",
        "checkTime": 1706745600000,
        "temperature": 36.5,
        "isTempNormal": true,
        "isHandNormal": true,
        "isPassed": true,
        "handStatus": "NORMAL",
        "healthCertStatus": "VALID",
        "symptomFlags": "",
        "remark": ""
      }
    ],
    "hasMore": true,
    "lastRecordId": 1100,
    "syncTime": 1706745600000
  }
}
```

**使用说明**

1. 首次同步：`lastRecordId=0`
2. 保存返回的 `lastRecordId`
3. 下次同步使用上次返回的 `lastRecordId`
4. 如果 `hasMore=true`，继续拉取

### 12. 账号信息同步接口

同步员工账号信息（登录账号）。

**请求**

```http
POST /api/users/sync
Authorization: Bearer {token}
Content-Type: application/json

{
  "action": "create",
  "users": [
    {
      "username": "001",
      "password": "123456",
      "passwordType": "plain",
      "employeeId": "001",
      "role": "employee",
      "status": "active"
    }
  ]
}
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| action | string | 是 | 操作类型：create/update/delete/sync |
| users | array | 是 | 用户列表 |
| username | string | 是 | 登录用户名（工号） |
| password | string | 是 | 密码 |
| passwordType | string | 否 | 密码类型：plain/md5/bcrypt（默认plain） |
| employeeId | string | 否 | 关联员工工号 |
| role | string | 否 | 角色：employee/admin（默认employee） |
| status | string | 否 | 状态：active/disabled（默认active） |

**操作类型说明**

| action | 说明 |
|--------|------|
| create | 新增（如果已存在则跳过） |
| update | 更新（如果不存在则跳过） |
| delete | 删除（如果不存在则跳过） |
| sync | 全量同步（先清空所有账号，再新增） |

**响应**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "received": 10,
    "success": 10,
    "failed": 0,
    "details": [
      {
        "username": "001",
        "status": "success",
        "message": "同步成功"
      }
    ]
  }
}
```

**错误码**

| 错误码 | 说明 |
|-------|------|
| 0 | 成功 |
| 2001 | 参数错误 |
| 2005 | 用户名已存在 |
| 2006 | 数据验证失败 |
| 2009 | 密码不能为空 |
| 2010 | 无效的角色 |
| 2011 | 无效的状态 |

**完整示例**

```json
// 新增账号
{
  "action": "create",
  "users": [
    {
      "username": "001",
      "password": "123456",
      "employeeId": "001",
      "role": "employee",
      "status": "active"
    },
    {
      "username": "002",
      "password": "123456",
      "employeeId": "002",
      "role": "employee",
      "status": "active"
    }
  ]
}

// 更新密码
{
  "action": "update",
  "users": [
    {
      "username": "001",
      "password": "newpassword888"
    }
  ]
}

// 禁用账号
{
  "action": "update",
  "users": [
    {
      "username": "001",
      "status": "disabled"
    }
  ]
}

// 删除账号
{
  "action": "delete",
  "users": [
    {
      "username": "001"
    }
  ]
}

// 全量同步
{
  "action": "sync",
  "users": [
    {
      "username": "001",
      "password": "123456",
      "employeeId": "001",
      "role": "employee"
    },
    {
      "username": "admin",
      "password": "admin888",
      "role": "admin"
    }
  ]
}
```

## 调用示例

### JavaScript 示例

```javascript
// 1. 登录获取 Token
const loginRes = await fetch('http://192.168.1.100:8080/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username: 'admin', password: '123456' })
});
const { data: { token } } = await loginRes.json();

// 2. 查询记录
const recordsRes = await fetch(
  'http://192.168.1.100:8080/api/records?startDate=2026-01-01&endDate=2026-01-31',
  {
    headers: { 'Authorization': `Bearer ${token}` }
  }
);
const { data } = await recordsRes.json();
console.log(data.list);
```

### Python 示例

```python
import requests

# 1. 登录
login_res = requests.post('http://192.168.1.100:8080/api/auth/login', json={
    'username': 'admin',
    'password': '123456'
})
token = login_res.json()['data']['token']

# 2. 查询记录
records_res = requests.get(
    'http://192.168.1.100:8080/api/records?startDate=2026-01-01&endDate=2026-01-31',
    headers={'Authorization': f'Bearer {token}'}
)
data = records_res.json()['data']
print(data['list'])
```

## 注意事项

1. **设备 IP**：实际使用时替换为设备的真实 IP 地址
2. **网络要求**：确保客户系统能访问设备的 8080 端口
3. **Token 有效期**：Token 有效期为 24 小时，过期后需重新登录
4. **时间戳**：所有时间戳单位为毫秒（Java 标准）
5. **分页建议**：单次查询建议不超过 100 条，大数据量请使用增量同步
6. **图片下载**：图片 URL 有效期与 Token 相同，需及时下载
7. **账号同步**：同步账号后，需要在设备上**重新登录**使新账号生效
8. **全量同步**：`action=sync` 会清空所有现有账号，请谨慎使用
