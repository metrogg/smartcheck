# 一、接口设计原则

| 原则 | 说明 |
|------|------|
| **Result 包装** | 所有方法返回 Result<T>，明确成功/失败 |
| **粗粒度** | 按"数据/功能领域"划分接口 |
| **KISS** | 简单够用，不过度设计 |
| **数据验证** | 关键操作需要参数校验 |
| **权限控制** | 敏感操作需要权限验证 |

# 三、统一错误码（AppError）

在定义接口之前，先定义统一的错误类型：

```kotlin
/**
 * 统一错误类型
 */
sealed class AppError {
    // ═══════════════════════════════════════════════════════════
    // 通用错误
    // ═══════════════════════════════════════════════════════════
    
    /** 资源不存在 */
    object NotFound : AppError()
    
    /** 数据验证失败 */
    data class ValidationError(
        val field: String,      // 字段名
        val message: String     // 错误信息
    ) : AppError()
    
    /** 重复数据 */
    data class DuplicateError(
        val field: String,       // 重复的字段
        val value: String       // 重复的值
    ) : AppError()
    
    /** 未授权 */
    object Unauthorized : AppError()
    
    /** 权限不足 */
    object Forbidden : AppError()
    
    // ═══════════════════════════════════════════════════════════
    // 硬件错误
    // ═══════════════════════════════════════════════════════════
    
    /** 硬件未初始化 */
    object HardwareNotReady : AppError()
    
    /** 硬件通信失败 */
    data class HardwareError(
        val device: String,      // 设备名
        val detail: String      // 错误详情
    ) : AppError()
    
    // ═══════════════════════════════════════════════════════════
    // AI 错误
    // ═══════════════════════════════════════════════════════════
    
    /** 模型未加载 */
    object ModelNotLoaded : AppError()
    
    /** 检测失败 */
    data class DetectionError(
        val type: String,        // 检测类型
        val message: String     // 错误信息
    ) : AppError()
    
    /** 未检测到目标 */
    object NoTargetDetected : AppError()
    
    // ═══════════════════════════════════════════════════════════
    // 系统错误
    // ═══════════════════════════════════════════════════════════
    
    /** 未知错误 */
    data class UnknownError(val message: String) : AppError()
}

/**
 * 错误转字符串
 */
fun AppError.toDisplayString(): String {
    return when (this) {
        is AppError.NotFound -> "数据不存在"
        is AppError.ValidationError -> "${field}: ${message}"
        is AppError.DuplicateError -> "${field} 已存在: ${value}"
        is AppError.Unauthorized -> "请先登录"
        is AppError.Forbidden -> "权限不足"
        is AppError.HardwareNotReady -> "硬件未就绪"
        is AppError.HardwareError -> "硬件错误: ${device} - ${detail}"
        is AppError.ModelNotLoaded -> "AI 模型未加载"
        is AppError.DetectionError -> "检测失败: ${message}"
        is AppError.NoTargetDetected -> "未检测到目标"
        is AppError.UnknownError -> "未知错误: ${message}"
    }
}
```

# 四、权限验证机制

在需要权限验证的接口中，使用 Token 进行验证：

### 4.1 Token 获取

```kotlin
// 登录成功后获取 Token
val token = adminAuthService.login(account, password)
// Token 存储在内存中，用于后续请求
```

### 4.2 Token 验证方式

```kotlin
// 方式 1：传入 Token 参数（显式验证）
suspend fun deleteUser(userId: Long, adminToken: String): Result<Unit> {
    // 验证 Token 是否有效
    if (adminToken != getCurrentToken()) {
        return Result.failure(AppError.Unauthorized)
    }
    // 执行删除
}

// 方式 2：使用内部验证（隐式验证）
suspend fun deleteUser(userId: Long): Result<Unit> {
    // 内部获取 Token 验证
    val token = getCurrentToken() ?: return Result.failure(AppError.Unauthorized)
    // 执行操作
}
```

### 4.3 建议：使用方式 2

**推荐使用内部验证方式**，因为：
1. 调用方不需要关心 Token 管理
2. 减少参数传递
3. 安全性更高（Token 不会在日志中暴露）

### 4.4 需要权限验证的操作

| 操作 | 需要权限 |
|------|----------|
| deleteUser | ✅ 需要登录 |
| deleteOldRecords | ✅ 需要登录 |
| deleteAllRecords | ✅ 需要登录 |
| createUser | ❌ 不需要 |
| updateUser | ❌ 不需要 |
| saveRecord | ❌ 不需要 |

### 11.2.3 活体检测使用示例

```kotlin
/**
 * 完整的人脸识别 + 活体检测流程
 */
class SeetaFace6RecognitionUseCase(
    private val engine: ISeetaFace6Engine,
    private val userRepository: IUserRepository
) {
    
    suspend fun recognizeWithLiveness(bitmap: Bitmap): Result<RecognitionResult> {
        // 1. 人脸检测
        val faces = engine.detectFaces(bitmap)
            .getOrElse { return Result.failure(it) }
        
        if (faces.isEmpty()) {
            return Result.failure(AppError.NoTargetDetected)
        }
        
        val face = faces.first()
        
        // 2. 质量评估（可选，但推荐）
        val quality = engine.assessQuality(bitmap, face).getOrNull()
        if (quality != null && !engine.isQualityPass(quality, config.qualityThreshold)) {
            return Result.failure(
                AppError.QualityBelowThreshold("人脸质量不达标")
            )
        }
        
        // 3. 活体检测 ⭐（重要！）
        val liveness = engine.detectLiveness(bitmap).getOrElse { 
            // 活体检测失败时，可以降级处理或拒绝
            return Result.failure(AppError.LivenessCheckFailed)
        }
        
        if (!engine.isLive(liveness)) {
            return Result.failure(
                AppError.NotLivePerson("活体检测未通过: ${liveness.label}")
            )
        }
        
        // 4. 提取特征
        val faceImage = cropFace(bitmap, face)
        val feature = engine.extractFeature(faceImage).getOrElse {
            return Result.failure(AppError.FeatureExtractFailed)
        }
        
        // 5. 与库中特征比对
        val user = userRepository.findByFaceFeature(feature)
        
        return Result.success(RecognitionResult(
            user = user,
            confidence = 0f,  // 实际应返回比对相似度
            livenessPassed = true,
            quality = quality
        ))
    }
}
```
