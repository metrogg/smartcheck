## 2. 晨检模块

### UC-101 人脸识别签到

| 项目 | 内容 |
|------|------|
| **用例编号** | UC-101 |
| **用例名称** | 人脸识别签到 |
| **参与者** | 员工 |
| **优先级** | 高 |
| **业务价值** | 核心晨检流程第一步 |
| **频率** | 每次晨检 |

**前置条件**
- 员工已录入系统

**主成功场景**
1. 员工点击"我要晨检"
2. 系统开启摄像头
3. 系统检测人脸
4. 系统比对人脸特征
5. 识别成功，显示员工信息（姓名、体温、健康证状态）
6. 流程结束

**扩展场景**
- 3a. 未检测到人脸
  - 3a1. 系统提示"未检测到人脸，请面向摄像头"
  - 3a2. 返回步骤3
- 4a. 人脸比对失败（库中无此员工）
  - 4a1. 系统提示"未找到匹配员工"
  - 4a2. 返回步骤3
- 4b. 健康证已过期
  - 4b1. 系统提示"健康证已过期，禁止上岗"
  - 4b2. 阻止晨检，流程结束

**异常场景**
- * 摄像头打开失败
  - 系统提示"摄像头异常，请联系管理员"

**后置条件**
- 员工身份已确认
- 显示员工信息和健康证状态

### UC-102 测量体温

| 项目 | 内容 |
|------|------|
| **用例编号** | UC-102 |
| **用例名称** | 测量体温 |
| **参与者** | 员工 |
| **优先级** | 高 |
| **业务价值** | 晨检必检项 |
| **频率** | 每次晨检 |

**前置条件**
- UC-101 人脸识别签到成功

**主成功场景**
1. 系统获取红外测温数据
2. 系统显示实时体温
3. 体温稳定后，系统记录体温值
4. 判断体温是否正常（≤37.5°C）
5. 流程结束

**扩展场景**
- 4a. 体温异常（>37.5°C）
  - 4a1. 系统显示"体温异常"
  - 4a2. 显示警告提示
  - 4a3. 继续流程（记录异常）

**后置条件**
- 体温已记录

### UC-103 手部检测

| 项目 | 内容 |
|------|------|
| **用例编号** | UC-103 |
| **用例名称** | 手部异物检测 |
| **参与者** | 员工 |
| **优先级** | 高 |
| **业务价值** | 晨检必检项 |
| **频率** | 每次晨检（手心+手背） |

**前置条件**
- UC-102 测量体温完成

**主成功场景**
1. 系统切换到手部摄像头
2. 员工伸右手手心
3. 系统检测手心
4. 系统记录手心照片（有框）
5. 员工翻转手背
6. 系统检测手背
7. 系统记录手背照片（有框）
8. 判断是否有异物
9. 流程结束

**扩展场景**
- 2a. 未检测到手
  - 2a1. 系统提示"请将手放入检测区域"
  - 2a2. 返回步骤2
- 8a. 检测到异物
  - 8a1. 系统显示异物类型和位置（框住异物）
  - 8a2. 系统提示"检测到异物，请处理后重测"
  - 8a3. 员工可选择重测（返回步骤2）或继续（记录异常）

**后置条件**
- 手部检测结果已记录
- 手部照片已保存

## 3. 员工管理模块

### UC-201 录入员工

| 项目 | 内容 |
|------|------|
| **用例编号** | UC-201 |
| **用例名称** | 录入员工 |
| **参与者** | 管理员 |
| **优先级** | 高 |
| **业务价值** | 员工入职必做 |
| **频率** | 新员工入职 |

**前置条件**
- 管理员已登录

**主成功场景**
1. 管理员点击"添加员工"
2. 填写员工基本信息（姓名*、工号*、身份证）
3. 拍摄员工人脸照片
4. 系统提取人脸特征
5. 拍摄员工健康证照片
6. 填写健康证有效期
7. 系统自动计算健康证剩余天数
8. 系统保存员工信息
9. 显示"添加成功"

**扩展场景**
- 3a. 工号已存在
  - 3a1. 系统提示"工号已存在"
  - 3a2. 返回步骤3
- 5a. 健康证有效期不合法
  - 5a1. 系统提示"结束日期不能早于开始日期"
  - 5a2. 返回步骤6

**异常场景**
- * 摄像头无法打开
  - 系统提示"摄像头异常，无法拍摄"

**后置条件**
- 员工信息已保存
- 人脸特征已提取
