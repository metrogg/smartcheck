# 🏗️ 新架构设计方案（方案 B：引入 UseCase 层）

## 一、老架构痛点分析

### 痛点 1：ViewModel 职责过重（900+ 行）

**现状：** MainViewModel 干了所有事情

```kotlin
// MainViewModel.kt 现状 - 900+ 行
class MainViewModel {
    // 1. UI 状态管理
    private val _uiState = MutableStateFlow(UiState())
    
    // 2. 业务逻辑（状态机）
    private fun onFaceRecognized(...) { ... }
    private fun startTemperatureMeasure() { ... }
    private fun performHandCheck() { ... }
    
    // 3. 硬件控制
    hardwareRepository.beep("success")
    hardwareRepository.openDoor()
    
    // 4. 文件操作
    FileUtil.saveBitmapToInternalResult(...)
    
    // 5. 语音播放
    voicePrompter.speak("欢迎")
    
    // 6. 数据保存
    recordRepository.insertRecord(record)
}
```

**问题：** 
- 违反单一职责原则
- 无法复用
- 无法测试（需要 Mock 整个世界）

---

### 痛点 2：Repository 无接口，无法测试

**现状：**

```kotlin
class MainViewModel @Inject constructor(
    private val hardwareRepository: HardwareRepository,  // 具体类
    private val voicePrompter: VoicePrompter,          // 具体类
    private val recordRepository: RecordRepository,      // 具体类
)
```

**问题：** 无法注入 Mock，单元测试几乎不可能

---

### 痛点 3：业务逻辑散落，无法复用

**现状：** 
- 晨检流程逻辑在 MainViewModel
- 如果另一个页面要用"测温+手检"流程？
- 只能复制粘贴代码

---

### 痛点 4：无状态枚举，业务规则不清晰

**现状：** 健康证状态、晨检状态混在代码里

```kotlin
// 散落在各处
if (healthCertDaysRemaining < 7) { ... }  // 警告
if (healthCertEndDate < now) { ... }       // 禁止
if (temperature > 37.5f) { ... }           // 体温异常
```

### 2.2 各层职责说明（中英文对照）

| 英文 | 中文 | 做什么 / What | 不做什么 / Not |
|------|------|---------------|----------------|
| **Screen** | 界面/页面 | 显示界面、处理用户点击 | 业务逻辑 |
| **ViewModel** | 视图模型 | 管理 UI 状态、分发事件 | 具体业务逻辑 |
| **UseCase** | 用例/业务逻辑 | 具体的业务流程 | 界面、数据库 |
| **Repository** | 仓库 | 统一管理数据访问 | 业务逻辑 |
| **Data Layer** | 数据源 | 实际存取数据 | 业务逻辑 |

### 2.3 核心 UseCase 设计

#### MorningCheckUseCase（晨检主编排）

```kotlin
class MorningCheckUseCase(
    private val userRepository: IUserRepository,
    private val recordRepository: IRecordRepository,
    private val hardwareService: IHardwareService,
    private val voiceService: IVoiceService,
    private val faceEngine: IFaceEngine,
    private val handDetector: IHandDetector
) {
    // 完整的晨检流程
    suspend fun startCheck(bitmap: Bitmap): CheckResult { ... }
    
    // 各个步骤
    suspend fun recognizeFace(bitmap: Bitmap): FaceResult { ... }
    suspend fun measureTemperature(): TemperatureResult { ... }
    suspend fun checkHand(bitmap: Bitmap): HandCheckResult { ... }
    suspend fun submitSymptoms(symptoms: List<String>): SubmitResult { ... }
    suspend fun saveRecord(result: CheckResult): Record { ... }
}
```

**好处：**
- **可复用**：任何页面需要"晨检流程"，直接调用
- **可测试**：Mock 所有依赖，测试业务逻辑
- **易维护**：业务逻辑集中在一处

### 2.6 ViewModel 变薄

**Before（900+ 行）：**

```kotlin
class MainViewModel {
    // 900 行，包含所有业务逻辑
}
```

**After（100+ 行）：**

```kotlin
class MainViewModel(
    private val morningCheckUseCase: MorningCheckUseCase
) {
    private val _uiState = MutableStateFlow(HomeUiState())
    val uiState: StateFlow<HomeUiState> = _uiState.asStateFlow()
    
    fun onEvent(event: HomeEvent) {
        when (event) {
            is HomeEvent.CameraFrame -> processFrame(event.bitmap)
            is HomeEvent.RetryFace -> retryFace()
            is HomeEvent.RetryHandPalm -> retryHandPalm()
            is HomeEvent.RetryHandBack -> retryHandBack()
            is HomeEvent.SubmitSymptoms -> submitSymptoms(event.symptoms)
        }
    }
    
    private fun processFrame(bitmap: Bitmap) {
        viewModelScope.launch {
            val result = morningCheckUseCase.recognizeFace(bitmap)
            _uiState.update { it.copy(state = result.toUiState()) }
        }
    }
}
```

## 三、解决痛点对照表

| 痛点 | 老架构 | 新架构 |
|------|--------|--------|
| ViewModel 过重 | 900 行，包含一切 | 100 行，只管 UI |
| 无法测试 | 依赖具体类 | 依赖接口，可 Mock |
| 业务无法复用 | 复制粘贴 | UseCase 可复用 |
| 状态散落 | 散落代码中 | 枚举集中定义 |
| 改数据源困难 | 到处改 | 只改 Repository |
