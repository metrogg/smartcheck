- 🚫 禁止单独升级 Kotlin

- ✅ 如需升级，需同步升级整个 Kotlin 生态
- ✅ 升级前需在测试环境验证

### 2.2 依赖管理

| 依赖类型 | 策略 | 示例 |

| **Compose BOM** | 锁定版本 | `2023.10.01` |

| **Room** | 锁定版本 | `2.6.1` |
| **第三方库** | 谨慎引入 | 需评审 |

## 三、技术选型标准

### 3.1 新增依赖规则

| 条件 | 要求 | 优先级 |
|------|------|--------|
| **必须引入** | 解决核心功能缺失 | P0 |
| **建议引入** | 显著提升开发效率 | P1 |
| **谨慎引入** | 非必要功能 | P2 |
| **禁止引入** | 重复功能、风险库 | P0 禁止 |

### 3.2 选型评审清单

引入新依赖前，必须评估：

| 评估项 | 最低要求 |
|--------|----------|
| **活跃度** | 最近 6 个月有更新 |
| **维护性** | 有明确维护团队 |
| **兼容性** | 支持当前 minSdk 26 |
| **体积** | 增量 < 1MB |
| **安全性** | 无已知漏洞 |
| **许可证** | 商用友好 (Apache 2.0 / MIT) |

### 3.3 推荐库清单

| 场景 | 推荐库 | 备选 |
|------|--------|------|
| **图片加载** | Coil | Glide |
| **JSON解析** | Kotlinx Serialization | Gson |
| **日志** | Timber | - |
| **权限** | Accompanist | - |
| **网络** | Retrofit + OkHttp | (预留，未来联网用) |
| **加密** | Android Security Crypto | - |

### 3.4 禁止使用清单

| 库 | 原因 | 替代方案 |
|----|------|----------|
| `kotlinx-coroutines:1.0` | 版本过旧 | 使用 bundled 版本 |
| `androidx.legacy:support*` | 已废弃 | 使用 AndroidX |
| `com.google.code.gson` | 非空安全 | Kotlinx Serialization |
| `androidx.appcompat:appcompat` | 非必要 | AndroidX Core |

## 四、架构规范

### 4.1 分层架构

```
┌─────────────────────────────────────────────┐
│  UI Layer (Compose + ViewModel)             │
├─────────────────────────────────────────────┤
│  UseCase Layer (业务逻辑)                   │
├─────────────────────────────────────────────┤
│  Repository Layer (接口 + 实现)              │
├─────────────────────────────────────────────┤
│  Data Layer (Room / File / Serial / JNI)   │
└─────────────────────────────────────────────┘
```

### 4.2 目录结构

```
app/src/main/java/com/smartcheck/app/
├── domain/                    # 🆕 Domain Layer
│   ├── model/                 # 领域模型
│   ├── usecase/               # UseCase
│   └── repository/           # 接口定义
│
├── data/                      # Data Layer
│   ├── repository/           # Repository 实现
│   ├── db/                   # Room
│   └── serial/               # 串口通信
│
├── ui/                        # UI Layer
│   ├── screens/
│   ├── components/
│   └── theme/
│
├── viewmodel/                # ViewModel
│
└── di/                       # Hilt Module
```

### 4.3 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | 小写 + 英文 | `com.smartcheck.app` |
| 类名 | PascalCase | `MainViewModel` |
| 函数名 | camelCase | `getUserById` |
| 常量 | UPPER_SNAKE | `MAX_RETRY_COUNT` |
| 接口 | I开头 + PascalCase | `IUserRepository` |
| UseCase | PascalCase + UseCase | `MorningCheckUseCase` |

## 五、代码规范

### 5.1 Kotlin 规范

| 规则 | 要求 | 优先级 |
|------|------|--------|
| **空安全** | 优先使用 `?.` `?:`, 避免 `!!` | 必须 |
| **类型推导** | 局部变量可省略类型 | 推荐 |
| **显式类型** | 公共API必须显式类型 | 必须 |
| **不可变** | 优先 `val` | 推荐 |
| **单表达式** | 简洁函数用 `= ` | 推荐 |
| **Trailing Comma** | 使用尾随逗号 | 必须 |

### 5.2 Compose 规范

| 规则 | 要求 |
|------|------|
| **Modifier** | 可重用的 Composable 必须接受 `modifier: Modifier = Modifier` |
| **State** | 使用 `remember` 和 `rememberSaveable` |
| **Effect** | 副作用使用 `LaunchedEffect` / `DisposableEffect` |
| **命名** | Composable 以大写开头 |

### 5.3 注释规范

| 场景 | 要求 |
|------|------|
| **公共API** | 必须添加 KDoc 注释 |
| **复杂逻辑** | 必须添加行注释 |
| **TODO** | 必须标注负责人和日期 |
| **禁止** | 禁止无意义注释（如 `// i++`) |

# 十三、开发实践规范（参考标准）

> 以下是我们值得参考学习的开发最佳实践。虽然当前团队规模可能暂时无法全部实施，但建议作为目标持续改进。

## 13.1 版本控制规范

### 分支管理策略

| 分支 | 用途 | 命名规则 | 生命周期 |
|------|------|----------|----------|
| `main` / `master` | 生产分支 | - | 永久 |
| `develop` | 开发分支 | - | 永久 |
| `feature/*` | 功能开发 | `feature/功能名` | 临时 |
| `fix/*` | Bug修复 | `fix/问题描述` | 临时 |
| `hotfix/*` | 紧急修复 | `hotfix/问题描述` | 临时 |
| `release/*` | 发布准备 | `release/版本号` | 临时 |

### Git 工作流

```
┌─────────────────────────────────────────────────────────┐
│                    GitFlow 工作流                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│    feature/xxx ──┐                                     │
│                  │                                     │
│    feature/yyy ──┼──► develop ──► release ──► main   │
│                  │         │            │              │
│    fix/zzz ──────┘         │            │              │
│                            │            │              │
│                    hotfix ──┴────────────┘              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 提交规范（Conventional Commits）

```
<类型>(<范围>): <描述>

# 示例
feat(user): 添加用户登录功能
fix(record): 修复记录查询分页问题
docs(api): 更新接口文档
refactor(useCase): 优化晨检流程
test(unit): 添加用户注册单元测试
chore(deps): 升级 Room 版本
```

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档更新 |
| `style` | 代码格式 |
| `refactor` | 重构 |
| `test` | 测试 |
| `chore` | 构建/依赖 |

### TDD 流程

```
┌────────────────────────────────────────────┐
│              TDD 循环                       │
├────────────────────────────────────────────┤
│                                            │
```

## 13.6 持续集成（CI）

### CI 流程

```
代码提交 → 自动构建 → 静态分析 → 单元测试 → 合并
    ↓
  失败 → 通知 → 修复
```

### GitHub Actions 示例

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up JDK 17
        uses: actions/setup-java@v3
        with:
          java-version: '17'
          
      - name: Grant execute permission
        run: chmod +x gradlew
        
      - name: Build
        run: ./gradlew assembleDebug
        
      - name: Run tests
        run: ./gradlew test
        
      - name: Static analysis
        run: ./gradlew lint
        
      - name: Build APK
        uses: actions/upload-artifact@v3
        with:
          name: app-debug
          path: app/build/outputs/apk/debug/app-debug.apk
```

### CI 检查项

| 检查项 | 工具 | 失败处理 |
|--------|------|----------|
| 编译 | Gradle | 阻止合并 |
| 单元测试 | JUnit | 阻止合并 |
| 静态分析 | Lint/Ktlint | 警告/阻止 |
| 插桩测试 | Espresso | 可选 |
